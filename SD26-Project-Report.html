1. Introduction

1.1 Project Overview

The SD26 Encoder is a secure Spring Boot web application that provides encoding and decoding of text using Hex (hexadecimal) and Base64 formats. The application is powered by the Apache Commons Codec library and demonstrates a comprehensive DevSecOps pipeline with formal verification, extensive testing, performance benchmarking, security scanning, and container orchestration.

1.2 Objectives

  • CI/CD: Fully automated build, test, and deployment pipeline using GitHub Actions
  • Formal Verification: JML specifications verified via OpenJML
  • Testing: 40 JUnit tests with 100% test strength in mutation analysis
  • Performance: JMH microbenchmarks measuring throughput
  • Security: Multi-layer security scanning (Snyk, GitGuardian, SonarCloud)
  • Containerization: Docker image available on DockerHub

3. Containerization (Docker)

✅ Requirement: A Docker image for the application ready to be orchestrated is available in DockerHub.

3.1 Docker Implementation

To ensure the application runs consistently across environments, we containerized the application using a multi-stage Dockerfile. The build stage compiles the application with Maven, and the runtime stage creates a minimal image based on eclipse-temurin:21-jre-alpine.

# Stage 1: Build
FROM eclipse-temurin:21-jdk-alpine AS builder
WORKDIR /build
COPY pom.xml src ./
RUN mkdir -p /app && apk add --no-cache maven && \
    mvn clean package -DskipTests -Djacoco.skip=true -q && \
    mv target/sd26-encoder.jar /app/sd26-encoder.jar

# Stage 2: Runtime (minimal)
FROM eclipse-temurin:21-jre-alpine
RUN addgroup -S sd26 && adduser -S sd26 -G sd26
WORKDIR /app
COPY --from=builder /app/sd26-encoder.jar ./
USER sd26
EXPOSE 8080
HEALTHCHECK CMD wget -qO- http://localhost:8080/actuator/health || exit 1
ENTRYPOINT ["java", "-jar", "/app/sd26-encoder.jar"]

Key security features:

  • Runs as non-root user (sd26)
  • Multi-stage build reduces image size to ~97 MB
  • Health check via /actuator/health endpoint
  • Minimal Alpine base image reduces attack surface

5. Formal Verification (JML)

✅ Requirement: The core methods of the application have a formal specification in JML, verified using OpenJML.

5.1 JML Specifications

All 4 core methods in CodecService.java are annotated with Java Modeling Language (JML) specifications defining strict preconditions (@requires) and postconditions (@ensures) to formally describe method contracts.

encodeHex(String input)

//@ public normal_behavior
//@ requires input != null;
//@ ensures \result != null;
//@ also
//@ public exceptional_behavior
//@ requires input == null;
//@ signals_only IllegalArgumentException;
public String encodeHex(String input) { ... }

decodeHex(String hexInput)

//@ public normal_behavior
//@ requires hexInput != null;
//@ ensures \result != null;
//@ also
//@ public exceptional_behavior
//@ requires hexInput == null;
//@ signals_only IllegalArgumentException;
public String decodeHex(String hexInput) { ... }

encodeBase64(String input)

//@ public normal_behavior
//@ requires input != null;
//@ ensures \result != null;
//@ also
//@ public exceptional_behavior
//@ requires input == null;
//@ signals_only IllegalArgumentException;
public String encodeBase64(String input) { ... }

decodeBase64(String base64Input)

//@ public normal_behavior
//@ requires base64Input != null;
//@ ensures \result != null;
//@ also

6.5 Web Application Security

✅ Requirement: The web application shows no vulnerabilities.

The following security measures are implemented in the web application:

  • Input Validation: Null checks on all methods, Base64 format validation before decoding, Hex format validation
  • XSS Protection: Thymeleaf auto-escapes HTML content (th:text), user input is never rendered as raw HTML
  • CSRF Protection: Spring Boot includes CSRF protection by default for all POST endpoints
  • Exception Handling: All encoding/decoding exceptions are caught and returned as user-friendly error messages
  • HTTP Hardening: Actuator endpoints restricted to health and info only, HTTP TRACE disabled
  • Secure Dependencies: All libraries upgraded to versions with no known critical vulnerabilities
  • Automated Scanning: Snyk, GitGuardian, and SonarCloud run on every commit
